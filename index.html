User Switching

Click the "Switch User" button to open a modal where you can:

  • Switch between visitor, random user, or custom user modes
  • Specify exact user_id and email for testing
  • Add custom attributes to the user
  • Provide JWT token for Identity Verification

The tool will properly shutdown any existing Intercom session and initialize a new one with your specified settings.


Getting 403 Errors?

If you're getting 403 Forbidden errors, your workspace likely requires Identity Verification. You'll need to:

  1. Generate a JWT token on your server using your workspace's Identity Verification secret
  2. Enter the JWT token in the "Identity Verification" field when switching users
  3. Make sure the user_hash in the JWT matches the user_id you're using

Learn more about Identity Verification


URL Parameters (Legacy)

You can still use URL hash parameters for initial setup:

  • app_id: Your Intercom app ID
  • user_id: User identifier
  • email: User email address

Example: #app_id=abc123&user_id=user456&email=test@example.com
